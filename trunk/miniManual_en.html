Download/Updates

The engine has 2 ways to do these processes:

From url

It downloads the archives from a url with wget and you can later give it parameters after the url.

Extracting link from a web page

Download the web page and search the link to download withing the HTML code.

Types of execution

All the types are based on executing the process and waiting for them to finish to later analize result, so only the differences are highlighted.

Normal

Reads the result of the file from the report. This method allows debug mode.

Piped

Reads the result that would show the scan on screen, if it doesn't read anything, it will show a report file. This method admits debug mode.

AntiBlock

Executes process from MSDOS, wait for it to finish, close the window and read the report file. Allows debug mode.

AntiWait

Once the scan is finished it sends the pulsation of a keystroke. DOES NOT admit debug mode.

Enigne variables

The program subtitutes these variables by the paths.

%ReportPath%

Path where the AV report is generated.

%EOL%

End Of Line, is used for delimiters to indicate end of line.

%FileToScan%

Entire path of file which would be scaned.

%ExtractKimsDir%

Path where engine is located. (KIMS.exe)

Some examples:

H0= --nombr /a /z --withtype=all --dmnoheur --nolinks --nostats /q --log="%ReportPath%" "%FileToScan%"

KIMS command line usage

/update

Updates the firms DB of last used AVs.

[sample path]

Scans and shows results on screen when finished as usual.

[sample path] [log path]

Scans in silent mode [sample  path] and generates a log with the results in [log path]
