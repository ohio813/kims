Características

·         Requerimientos:

Windows XP/Vista/7

·         Ventajas:

1.      Automatización de todas las tareas.

2.      Fácilmente configurable.

3.      Soporte de línea de comando para las principales tareas.

·         Desventajas:

1.      El análisis es solo en scan time.

2.      El límite máximo de peso de las configuraciones y lenguajes es de 64K.

Descargas/Actualizaciones

El motor tiene 2 formas de hacer estos procesos:

Desde URL

Se baja el archivos desde una URL con wget por lo que después de la URL se le pueden mandar parámetros.

Extrayendo link de una pagina Web

Baja la página Web y busca el link de descarga dentro del código HTML.

Tipos de ejecución

Todos los tipos se basan en ejecutar el proceso y aguardar a que terminen para después analizar el resultado, así que solo se aclararan las diferencias.

Normal

Lee el resultado desde el archivo de reporte. Este método admite modo debug.

Piped

Lee el resultado que mostraría el escaneo en pantalla y si no llegara a leer nada procesa el del archivo de reporte. Este método admite modo debug.

AntiBlock

Ejecuta proceso de MSDOS, aguarda a que termine, cierra la ventana y lee el archivo de reporte. Este método admite modo debug.

AntiWait

Una vez terminado el escaneo se envía la pulsación de una tecla. NO admite modo debug.

Variables del motor

El programa sustituye estas variables por las rutas.

%ReportPath%

Ruta donde se genera el reporte del AV.

%EOL%

End Of Line, se usa para los delimitadores para indicar el final de línea.

%FileToScan%

Ruta entera del archivo que será escaneado.

%ExtractKimsDir%

Ruta donde esta el motor (KIMS.exe).

Algunos ejemplos:

H0= --nombr /a /z --withtype=all --dmnoheur --nolinks --nostats /q --log="%ReportPath%" "%FileToScan%"

Uso de KIMS por linea de comando

/update

Actualiza las bases de datos de firmas de los ultimos AVs usados.

[Ruta de muestra]

Escanea y cuando termina de hacerlo muestra los resultados en pantalla como hace normalmente.

[Ruta de muestra] [Ruta de log]

Escanea de forma silenciosa [Ruta de muestra] y genera un log con los resultados en [Ruta de log]
